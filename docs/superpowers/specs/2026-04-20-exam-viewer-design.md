# 시험 답안 Viewer 설계

작성일: 2026-04-20

## 1. 목표

폴더 안에 계속 추가될 시험 답안/해설 `md` 파일을 기반으로, 과목별 및 시험 종류별로 탐색 가능한 정적 HTML viewer를 만든다.

사용자는 다음을 할 수 있어야 한다.

- 과목을 선택한다.
- 해당 과목의 `중간/기말` 시험을 선택한다.
- 공개 시간이 지난 시험만 입장한다.
- 시험 상세에서 `정답표`와 `문항별 해설`을 탭으로 본다.
- 현재 서울 시각과 공개까지 남은 카운트다운을 확인한다.

관리자는 비밀번호 `RR#890rr`를 입력하면 현재 브라우저 세션 동안 공개 제한을 무시하고 모든 시험에 입장할 수 있어야 한다.

## 2. 핵심 제약

- 실행 방식은 `index.html` 더블클릭을 기본으로 한다.
- 나중에는 Git 기반 정적 배포 환경에서도 동일하게 동작해야 한다.
- 따라서 viewer는 브라우저 런타임에 로컬 `md` 파일을 직접 읽지 않는다.
- 시험 원본 `md`는 사전 생성 단계에서 viewer용 정적 데이터로 변환한다.
- 공개 판정 기준 시간대는 `Asia/Seoul` 고정이다.
- 공개 단위는 `md` 파일 1개 기준이다. 시험 하나 전체가 하나의 공개 시각을 가진다.

## 3. 권장 구조

프로젝트는 아래 3계층 구조를 가진다.

### 3.1 원본 콘텐츠

원본 시험 파일은 `md`로 유지한다. 각 파일은 YAML front matter를 가져야 한다.

예시:

```md
---
title: "2026학년도 1학기 뇌과학개론 중간고사 정답 및 해설"
subject: "뇌과학개론"
exam: "midterm"
releaseAt: "2026-04-21T09:00:00+09:00"
timezone: "Asia/Seoul"
year: 2026
term: "1학기"
---
```

필수 필드:

- `title`
- `subject`
- `exam`
- `releaseAt`

권장 필드:

- `timezone`
- `year`
- `term`

`exam` 값은 우선 `midterm` 또는 `final`로 표준화한다.

### 3.2 생성 스크립트

생성 스크립트는 모든 시험 `md`를 읽어서 viewer 전용 데이터 파일을 만든다.

역할:

- YAML front matter 파싱
- 본문 Markdown 파싱
- `정답표` 섹션과 `문항별 해설` 섹션 분리
- viewer 렌더링용 HTML 또는 사전 가공 데이터 생성
- 최종 결과를 정적 데이터 파일로 출력

권장 출력 대상:

- `data/exams.js` 또는
- `data/exams.json`

더블클릭 실행 안정성을 고려하면 `data/exams.js` 형태가 가장 단순하다.

### 3.3 정적 Viewer

정적 viewer는 아래 자산만으로 동작한다.

- `index.html`
- `app.js`
- `styles.css`
- 생성된 `data/exams.js`

viewer는 생성된 데이터만 읽고 다음만 담당한다.

- 목록 렌더링
- 공개 여부 계산
- 서울 시계 표시
- 카운트다운 표시
- 관리자 세션 우회
- 탭 전환

## 4. 화면 흐름

### 4.1 홈 화면

- 과목 카드 목록 표시
- 각 카드에 과목명과 포함된 시험 수 표시 가능

### 4.2 과목 상세 화면

- 선택한 과목의 시험 목록 표시
- `중간/기말` 시험 카드 표시
- 시험 카드별 상태 표시

공개 전 카드:

- `잠김` 상태 배지
- 공개 시각
- 현재 서울 시각
- 카운트다운
- 관리자 열기 버튼

공개 후 카드:

- `입장` 버튼

### 4.3 시험 상세 화면

- 시험 제목
- 과목/시험 종류 정보
- 공개 상태 또는 관리자 우회 상태
- 상단 탭 2개
  - `정답표`
  - `문항별 해설`

탭은 Markdown 원본을 그대로 노출하지 않고, 생성 단계에서 분리된 콘텐츠를 표시한다.

## 5. 데이터 모델

viewer에 공급되는 시험 단위 데이터는 최소 아래 필드를 가진다.

- `id`
- `title`
- `subject`
- `exam`
- `releaseAt`
- `timezone`
- `year`
- `term`
- `answerKeyHtml`
- `explanationsHtml`
- 선택적으로 `summaryHtml`

`isReleased`는 정적 데이터에 저장하지 않고 브라우저에서 계산한다.

## 6. 공개 판정 및 시계

### 6.1 판정 규칙

- 기준 시간대는 항상 `Asia/Seoul`
- 현재 시각과 `releaseAt`를 비교해 공개 여부 계산
- 공개 시각 도달 전에는 상세 화면 진입 제한
- 공개 시각 도달 즉시 UI 상태 자동 갱신

### 6.2 시계와 카운트다운

시험 카드 또는 상단 상태 영역에 다음을 표시한다.

- 현재 서울 시각
- 공개 시각
- 남은 시간

카운트다운은 1초 단위로 갱신한다.

## 7. 관리자 우회

### 7.1 동작 방식

- 비밀번호: `RR#890rr`
- 한 번 인증하면 현재 브라우저 세션 전체에 적용
- 잠긴 모든 시험을 즉시 열 수 있음

### 7.2 저장 방식

- `sessionStorage` 사용
- 같은 탭에서 새로고침 시 유지
- 브라우저 세션 종료 시 해제

### 7.3 보안 전제

이 기능은 정적 프런트엔드 환경에서의 운영 편의 장치다.
강한 보안을 제공하지 않으며, 접근 제어 수준의 기능으로 간주한다.

## 8. 생성 스크립트 규약

생성 스크립트는 다음을 보장해야 한다.

- 필수 front matter 누락 시 명확한 오류 출력
- `releaseAt` 파싱 실패 시 명확한 오류 출력
- `정답표`와 `문항별 해설` 섹션을 찾지 못하면 경고 또는 실패 처리
- 여러 `md` 파일을 일관된 정렬 규칙으로 출력

권장 입력 규칙:

- 본문 섹션 제목은 가능한 한 일관되게 유지
- `정답표`와 `문항별 해설` 제목 변형이 생길 수 있으면 허용 패턴 정의

## 9. 오류 처리

### 9.1 생성 단계

- 잘못된 파일은 조용히 무시하지 않는다.
- 어떤 파일이 왜 실패했는지 출력한다.
- 필요하면 전체 실패로 처리해 데이터 일관성을 지킨다.

### 9.2 Viewer 단계

- 데이터가 비어 있으면 안내 문구 표시
- 시험이 없으면 `표시 가능한 시험이 없습니다` 표시
- 관리자 비밀번호가 틀리면 즉시 오류 메시지 표시

## 10. 검증 기준

구현 완료 후 아래가 만족되어야 한다.

- 잠긴 시험과 공개된 시험이 올바르게 구분된다.
- 서울 시계가 표시된다.
- 카운트다운이 1초 단위로 감소한다.
- 공개 시각 도달 시 잠금 상태가 자동 해제된다.
- 관리자 인증 후 잠긴 시험도 즉시 열 수 있다.
- 과목 선택 후 해당 과목의 `중간/기말` 시험이 보인다.
- 시험 상세에서 `정답표`와 `문항별 해설` 탭 전환이 정상 동작한다.
- 새 `md` 추가 후 생성 스크립트를 다시 실행하면 목록에 반영된다.
- `index.html` 더블클릭 환경과 정적 배포 환경에서 모두 동작한다.

## 11. 구현 방향 요약

권장 구현 방향은 다음과 같다.

1. 기존 시험 `md`에 YAML front matter를 도입한다.
2. 생성 스크립트를 만들어 `data/exams.js`를 만든다.
3. 정적 viewer를 구현해 과목 목록, 시험 목록, 잠금 상태, 상세 탭을 표시한다.
4. 관리자 세션 우회와 서울 시계/카운트다운을 추가한다.
5. 새 `md`가 추가될 때 생성 스크립트만 다시 실행하면 반영되도록 운영한다.
